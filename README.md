Ответы
```
Роли FSMO (Flexible Single Master Operations)
Всего 5 ролей FSMO:

Роли на уровне леса (одна для всего леса):

Schema Master - отвечает за изменения схемы Active Directory

Domain Naming Master - управляет созданием и удалением доменов в лесу

Роли на уровне домена (в каждом домене):

PDC Emulator - эмулирует главный контроллер домена для устаревших систем, синхронизирует время, обрабатывает изменения паролей

RID Master - предоставляет пулы уникальных RID (Relative ID) для создания объектов безопасности

Infrastructure Master - обновляет ссылки на объекты из других доменов

2. GPO (Group Policy Object)
Что такое GPO:
Объект групповой политики - механизм централизованного управления настройками пользователей и компьютеров в домене.

Для чего нужна:

Управление безопасностью

Настройка рабочей среды

Установка программного обеспечения

Ограничение функциональности

Автоматизация задач

Состав GPO:

GPC (Group Policy Container) - хранится в Active Directory

GPT (Group Policy Template) - хранится в папке SYSVOL

Настройки делятся на:

Computer Configuration (настройки компьютера)

User Configuration (настройки пользователя)

3. Назначение программ
cmd.exe - командная строка Windows для выполнения команд

ping.exe - проверка доступности сетевого узла по ICMP

nslookup.exe - диагностика DNS, запросы к DNS-серверам

telnet.exe - подключение к удаленным серверам по протоколу Telnet

tracert.exe - трассировка маршрута пакетов в сети

mmc.exe - консоль управления Microsoft для оснасток

sfc.exe - проверка и восстановление системных файлов Windows

fixboot.exe - восстановление загрузочного сектора (в Windows XP)

checkdisk.exe (chkdsk) - проверка диска на ошибки файловой системы

4. Типы групп в Active Directory
Типы групп:

Группы безопасности - для назначения прав доступа

Группы рассылки - только для электронной почты

Подробнее о группах безопасности:

Локальные группы - только на локальном компьютере

Глобальные группы - для организации пользователей с одинаковыми требованиями к доступу

Универсальные группы - могут содержать пользователей и группы из любого домена леса

5. Области действия групп
Три области действия:

Domain Local - для назначения прав доступа к ресурсам в домене

Global - для организации пользователей внутри домена

Universal - для организации групп между доменами в одном лесу

6. Репликация контроллера домена
Что такое репликация:
Процесс синхронизации данных между контроллерами домена в рамках одного домена или леса.

Для чего нужна:

Обеспечение отказоустойчивости

Распределение нагрузки

Географическое распределение

Гарантия актуальности данных на всех контроллерах

Типы репликации:

Внутрисайтовая - быстрая, между контроллерами в одном сайте

Межсайтовая - сжатая, между контроллерами разных сайтов
```

Что такое DHCP?		
Что такое DNS и для чего он нужен?
Какие основные типы DNS записей вы знаете и для чего они?		
Что такое VLAN и для чего он нужен?		
Даны следующие маски подсетей. Сколько хостов может быть в каждой из них:
255.255.255.0 - 24 маска
255.255.255.128 - 25 маска
255.255.255.252 - 30 маска
255.255.255.254 - 31 маска
255.255.255.255 - 32 маска		

Ответы
```
Порты для служб:

DNS: 53 (TCP и UDP)

HTTPS: 443 (TCP)

Почтовый сервис: SMTP (отправка) - 25, POP3 (прием) - 110, IMAP (прием) - 143. Также для безопасных версий: SMTPS - 465, IMAPS - 993, POP3S - 995.

NTP: 123 (UDP)

NETBIOS: 137 (UDP - служба имен), 138 (UDP - служба датаграмм), 139 (TCP - сессии)

SSH: 22 (TCP)

FTP: 21 (TCP - управление) и 20 (TCP - данные) или в пассивном режиме случайные порты.

DHCP (Dynamic Host Configuration Protocol) - это протокол, который автоматически назначает IP-адреса и другие сетевые параметры (маску подсети, шлюз, DNS-серверы) устройствам в сети. Это избавляет от необходимости настраивать каждый компьютер вручную.

DNS (Domain Name System) - это система, которая преобразует доменные имена (например, example.com) в IP-адреса (например, 93.184.216.34). Она нужна для того, чтобы людям не приходилось запоминать числовые IP-адреса.

Основные типы DNS записей:

A (Address) - сопоставляет доменное имя с IPv4-адресом.

AAAA (IPv6 Address) - сопоставляет доменное имя с IPv6-адресом.

CNAME (Canonical Name) - создает псевдоним для другого доменного имени.

MX (Mail Exchange) - указывает серверы, обрабатывающие почту для домена.

TXT (Text) - используется для хранения текстовой информации, часто для проверки владения доменом, SPF, DKIM и т.д.

NS (Name Server) - указывает на DNS-серверы, ответственные за домен.

PTR (Pointer) - используется для обратного DNS-запроса (из IP в имя).

SOA (Start of Authority) - содержит информацию о зоне: первичный DNS-сервер, контакт администратора, тайминги обновления.

VLAN (Virtual Local Area Network) - это виртуальная локальная сеть, которая позволяет логически разделить физическую сеть на несколько изолированных широковещательных доменов. VLANы используются для:

Повышения безопасности путем изоляции трафика.

Уменьшения широковещательного трафика.

Упрощения управления сетью.

Группировки устройств по функциям, а не по местоположению.

Количество хостов в подсетях (учитываем, что два адреса зарезервированы: сетевой и широковещательный, кроме /31 и /32, где есть особенности):

/24 (255.255.255.0): 256 адресов, 254 хоста (256-2)

/25 (255.255.255.128): 128 адресов, 126 хостов (128-2)

/30 (255.255.255.252): 4 адреса, 2 хоста (4-2)

/31 (255.255.255.254): 2 адреса, 2 хоста (согласно RFC 3021, точка-точка линки, без широковещательного адреса)

/32 (255.255.255.255): 1 адрес, 1 хост (адрес узла)
```

Ответы
```
Я отвечу только на один вопрос.
RAID 0 (Stripping) - чередование данных без избыточности

Минимально: 2 диска

Преимущества: высокая скорость

Недостатки: нет отказоустойчивости

RAID 1 (Mirroring) - зеркалирование данных

Минимально: 2 диска

Преимущества: отказоустойчивость, хорошая скорость чтения

Недостатки: 50% потерь емкости

RAID 5 - чередование с распределенной четностью

Минимально: 3 диска

Преимущества: отказоустойчивость при потере 1 диска, эффективное использование емкости

Недостатки: медленная запись

RAID 6 - чередование с двойной четностью

Минимально: 4 диска

Преимущества: отказоустойчивость при потере 2 дисков

Недостатки: еще более медленная запись

RAID 10 - зеркалирование + чередование

Минимально: 4 диска

Преимущества: высокая производительность и отказоустойчивость

Недостатки: 50% потерь емкости


```
